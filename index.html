Food Pal

For my fourth year software architecture and design course, I spent a semester working in a group working on an Android grocery tracking app that allows users to create grocery lists, add these groceries to their pantry, be notified when their items are going to expire and determine any potential recipes that the user wants to create with ingredients in their pantry.


Functionality Included

  • The pantry allows the user to add grocery items in their pantry
  • The pantry list shows a list of grocery items with its name, date of purchase and date of expiry. It also shows items close to expiry date to remind the user.
  • Pantry item: velocity of consumption is tracked, and we pre-emptively remind the user if they’re about to run out of this ingredient.
  • If the user clicks on a pantry item, they will be navigated to a screen which shows the basic information of this item (name, quantity, purchased date, expiry date) as well as all the recipes that use this item as an ingredient.
  • When any pantry item is about to expire, the user is sent a notification (pull down top menu bar).
  • The Recipe page shows a list of recipes. A user can look at all the recipes available or specifically look into what recipes they can make with their current pantry
  • The recipes and it will display the ingredients and instructions for that recipe. The user can easily add all or specific ingredients to their grocery lists
  • The grocery cart allows users to add, delete and search grocery lists in their cart. They can view or make changes to grocery items in individual grocery lists. Users can also all the grocery items on this list into the pantry by selecting the “Add to Pantry” button. *Note: the app uses real dataConcurrency

Fantasy Hockey Trade Simulator

In my spare time, I run an intense fantasy hockey league. Due to some arguing amongst the GMs of what trades should or should not be a veto, I created a trade analyzer to evaluate the trade and if the value from each side is within a specifc range then determine whether trades should be approved or not.

This trade analyzer is a command-line tool, built with Python that takes in player stats from Yahoo's Fantasy API and pre-season projections excel sheet from @domluszczyszyn. With these stats, it adjusts for age, injury-riddled seasons and value above replacement (i.e. free agent level player) for each position to calculate a player's RichVal.
